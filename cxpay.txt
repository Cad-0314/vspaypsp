1、API Interface Description
Request method
All interface Request methods are post methods that can be submitted using JSON
or form
Signature
The general steps for signature generation are as follows: The first step is to set all the data sent or received as set M, remove the sign
parameter and the null parameter to obtain set N. Sort all the parameters in set N from
small to large according to the ASCII code of the parameter name (dictionary order), and
use the format of URL key value pairs (i.e. key1=value1&key2=value2...) to concatenate
them into a string A. Step 2, concatenate the string A with&key=bHXJGsw6CsxkSb... to obtain the
stringSignTemp string, stringSignTemp=”
account=account&amount=amount&mchId=mchId&notifyUrl=notifyUrl&orderNo=order
No&otherData=otherData&passageId=passageId&userName=userName&key=34797ffe
c7814a...”
Step three, perform MD5 operation on stringSignTemp, and then convert all
characters of the resulting string to lowercase to obtain the sign value signValue, sign=MD5(stringSignTemp).toLowerCase()
The data format returned by the interface
{
"code": 200,
"success": true,
"msg": null,
"desc": null,
"data":.....
}

code int Request return code, see data dictionary for values

desc string Error message will only appear if the request processing fails data object Request to return business data

Brief description
 After confirming the success or failure of collection or payment, the
merchant will be notified asynchronously several times. When the merchant
system receives peer-to-peer communication from the platform, it returns
the lowercase string 'success', indicating successful reception. Otherwise, the platform will repeatedly push notifications to the merchant, with a total
of 8 pushes within 40 minutes
Request URL
 The notifyURL received by the merchant for collection or
payment

 Content-Type: application/json
parameter
