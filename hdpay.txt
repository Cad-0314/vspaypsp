API documentation

0. Introduction
0.1. Purpose of Writing


1. The purpose of this document is to provide developers with the interface specifications required for development.

2. This document can be used as a reference for developers and testers.

0.2. Instructions for use


1. HTTPS is used for data transmission.

2. The character encoding is uniformly application/json UTF-8.

3. A response status code of 200 indicates a successful request and does not indicate the order status.

4. The unit of measurement is rupees (yuan).


0.3. Signature Instructions


The data to be signed is signed as follows:

1. Sort the parameter key values ​​in ascending ASCII order

. 2. Concatenate the parameter values ​​into a string stringA in the format key=value&key=value... (empty values ​​are not included in the signature concatenation).

3. Encrypt the above string using the platform's key using MD5: MD5(stringA&key=key).

4. Finally, convert the MD5-encrypted string to lowercase.
