# Interface Description


# F2PAY API Documentation


---


## API Conventions


1. **Content-Type**: All POST requests must use `application/json`.
2. **Character Encoding**: UTF-8 encoding is mandatory.
3. **Data Format**: Both request and response data must be in JSON format.
4. **Signature Algorithm**: `OPENSSL_ALGO_SHA256`.
5. **Amount Precision**: Amounts must be rounded to **two decimal places**.


---


## Important Notes


**Please carefully read the following notes to avoid financial losses due to integration issues.**


### 1. IP Configuration
- Add your server IP address in the merchant backend under `Setting -> Configuration -> Api Key`.
- Add our server IP address to your whitelist.


### 2. Response Handling
- All API responses include a `code` field. Any value other than `0000` indicates a failed request.
- The failure reason and the message displayed to the user should be encapsulated based on the `status` enumeration.
- **Do not directly display the `msg` field from the API response to the user** (it may contain sensitive information).


### 3. Currency
- The system automatically switches **currencies** based on the API used.


### 4. Payin Callbacks(Very important)
- **When the player’s order amount does not match the actual payment amount, the order state will be `UnequalPaid`. By default, we do not send callbacks in this case. If you need callbacks to be sent for `UnequalPaid` orders, please make sure to process the `actualAmount` field from the callback API, and contact our operations team to enable the callback for `UnequalPaid` status.**


- **For payin callbacks, regardless of whether the state is `Paid` or `UnequalPaid`, the player's balance should be credited based on the `actualAmount` field.**




### 5. Payin Request Frequency
- For the same user and the same payin amount, avoid repeatedly calling the payin API within a short time frame.


### 6. Payout Order Creation
- When using the **payout** order creation API, always check the response status `code`.
- If the `code` is not `0000`, consult our operations team to confirm the **channel status** before resubmitting.
- If the request times out or encounters a network error, consult our operations team to confirm the **channel status** before resubmitting.


### 7. Response Handling Logic
- Check the protocol field response.
- Check the business response.
- Check the transaction status.


### 8. How to test callbacks
- Please send the order number to the group and @ our technicians, we will manually call back for testing


### 9. Risk Control
- **Customer Name + Phone Number + Email + Amount** is used to determine different customer requests.
- Avoid hardcoding the name, phone number, and email, as this may lead to **risk control misjudgments**.

# Signature & Verification Protocol


## Request Parameters
| Parameter    | Field Name         | Definition          | Length | Required | Description                              |
|:------------|:---------------------|:--------------------|:-------|:---------|:-----------------------------------------|
| traceId     | Business Trace ID    | String              | 512    | Y        | Randomized alphanumeric string,Unique for each request          |
| signType    | Signature Type       | String              | 512    | Y        | Supported: RSA                           |
| merchantId  | Merchant ID          | String              | 32     | Y        |                                          |
| sign        | Signature Result     | String              | 512    | Y        | Request signature                        |
| bizContent  | Request Data         | String              | 512    | Y        | JSON string of request payload (see API docs) |


### Notes
1. All request structures follow this schema
2. `bizContent` must be JSON string representation of API-specific objects


---


## Response Parameters
| Parameter    | Field Name         | Definition          | Length | Required | Description                              |
|:------------|:---------------------|:--------------------|:-------|:---------|:-----------------------------------------|
| code        | Response Code        | String              | 16     | Y        | See response code list                   |
| msg         | Response Message     | String              | 512    | N        | Diagnostic information                   |
| sysTime     | System Timestamp     | String              | 24     | Y        | Format: `yyyyMMddHHmmssSSS`              |
| sign        | Signature Result     | String              | 512    | Y        | Response signature                       |
| bizContent  | Response Data        | JSON                |        | N        | JSON string of business data (see API docs) |


### Notes
3. All responses/callbacks follow this structure


---


## Signature & Verification
- **Signing**: Generate signature with merchant private key  
  `sign = RSAUtils.createSign(bizContent, 'merchant_private_key')`
  
- **Verification**: Validate signature with platform public key  
  `isValid = RSAUtils.verifySign(bizContent, 'platform_public_key')`


---


## Critical Rules
1. Signature original string (`stringA`) uses raw parameter names/values **without URLEncode**
2. Algorithm: `OPENSSL_ALGO_SHA256` (Sign with merchant private key, verify with platform public key)
3. This implements RSA signature creation, **not** RSA encryption

    public static String encrypt(String data, String privateKey) {
        // Base64 --> Key
        try {
            byte[] bytes = Base64.getDecoder().decode(privateKeyBase64);
            PKCS8EncodedKeySpec keySpec = new PKCS8EncodedKeySpec(bytes);
            KeyFactory keyFactory;
            keyFactory = KeyFactory.getInstance("RSA");
            PrivateKey privateKey = keyFactory.generatePrivate(keySpec);
            // Sign
            Signature signature = Signature.getInstance("SHA256WithRSA");
            signature.initSign(privateKey);
            signature.update(data.getBytes("UTF-8"));
            return Base64.getEncoder().encodeToString(signature.sign());
        } catch (Exception e) {
            log.error("Exception", e);
            return null;
        }
    }


    public static boolean verify(String data, String publicKey, String sign) {
        // Base64 --> Key
        try {
            byte[] bytes = Base64.getDecoder().decode(publicKeyBase64);
            X509EncodedKeySpec keySpec = new X509EncodedKeySpec(bytes);
            KeyFactory keyFactory;
            keyFactory = KeyFactory.getInstance("RSA");
            PublicKey publicKey = keyFactory.generatePublic(keySpec);
            // verify
            Signature signature = Signature.getInstance("SHA256WithRSA");
            signature.initVerify(publicKey);
            signature.update(data.getBytes("UTF-8"));
            return signature.verify(Base64.getDecoder().decode(sign));
        } catch (Exception e) {
            log.error("Exception", e);
            return false;
        }
    }

    private function encrypt($data, $privateKey) {
        $privateKeyResource = openssl_pkey_get_private($privateKey);
        if (!$privateKeyResource) {
            error_log("Invalid private key.");
            return null;
        }
        
        openssl_sign($data, $signature, $privateKeyResource, OPENSSL_ALGO_SHA256);
        return base64_encode($signature);
    }


    private function verify($data, $publicKey, $sign) {
        $publicKeyFormatted = "-----BEGIN PUBLIC KEY-----\n" . wordwrap($publicKey, 64, "\n", true) . "\n-----END PUBLIC KEY-----";

        if (!$publicKeyResource) {
            error_log("Invalid public key.");
            return false;
        }


        $decodedSign = base64_decode($sign);
        return openssl_verify($data, $decodedSign, $publicKeyResource, OPENSSL_ALGO_SHA256) === 1;
    }

    public static String encrypt(String data, String privateKey) {
        // Base64 --> Key
        try {
            byte[] bytes = Base64.getDecoder().decode(privateKeyBase64);
            PKCS8EncodedKeySpec keySpec = new PKCS8EncodedKeySpec(bytes);
            KeyFactory keyFactory;
            keyFactory = KeyFactory.getInstance("RSA");
            PrivateKey privateKey = keyFactory.generatePrivate(keySpec);
            // Sign
            Signature signature = Signature.getInstance("SHA256WithRSA");
            signature.initSign(privateKey);
            signature.update(data.getBytes("UTF-8"));
            return Base64.getEncoder().encodeToString(signature.sign());
        } catch (Exception e) {
            log.error("Exception", e);
            return null;
        }
    }


    public static boolean verify(String data, String publicKey, String sign) {
        // Base64 --> Key
        try {
            byte[] bytes = Base64.getDecoder().decode(publicKeyBase64);
            X509EncodedKeySpec keySpec = new X509EncodedKeySpec(bytes);
            KeyFactory keyFactory;
            keyFactory = KeyFactory.getInstance("RSA");
            PublicKey publicKey = keyFactory.generatePublic(keySpec);
            // verify
            Signature signature = Signature.getInstance("SHA256WithRSA");
            signature.initVerify(publicKey);
            signature.update(data.getBytes("UTF-8"));
            return signature.verify(Base64.getDecoder().decode(sign));
        } catch (Exception e) {
            log.error("Exception", e);
            return false;
        }
    }
